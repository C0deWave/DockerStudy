도커 공부

우선은 명령어를 간단히 정리해 보았다.

commit 규칙
1. Feat : 새로운 기능에 대한 커밋
2. Fix : 버그 수정에 대한 커밋
3. Build : 빌드 관련 파일 수정에 대한 커밋
4. Chore : 그 외 자잘한 수정에 대한 커밋
5. Ci : CI관련 설정 수정에 대한 커밋
6. Docs : 문서 수정에 대한 커밋
7. Style : 코드 스타일 혹은 포맷 등에 관한 커밋
8. Refactor :  코드 리팩토링에 대한 커밋
9. Test : 테스트 코드 수정에 대한 커밋
